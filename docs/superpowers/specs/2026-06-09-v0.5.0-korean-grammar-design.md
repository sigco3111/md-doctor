## 2. 아키텍처

### 2.1 모듈 구조

```
src/md_doctor/
├── checks/
│   └── korean_grammar.py        # [신규] KoreanGrammarCheck
└── cli.py                       # [수정] --korean 플래그
```

**왜 단일 모듈?**
- 5개 규칙이 모두 정규식 기반
- 복잡도 낮음, 단일 책임 유지

### 2.2 의존성 그래프

```
checks/korean_grammar.py  ← models, re (stdlib only)
cli.py                    ← checks.korean_grammar (--korean 등록)
```

순환 없음. `re` 만 사용.

### 2.3 활성화 메커니즘

- `korean-grammar` 체크는 `default_checks()` 에 **등록하지 않음** (기본 비활성)
- CLI `--korean` 플래그 시 명시적으로 체크를 `registry` 에 추가
- `korean-grammar` 단독: `md-doctor --korean README.md --checks korean-grammar`
- `korean-grammar` + 다른 검사: 모두 명시

또는 v0.4.0 fix 플래그와 같이 **opt-in 자동 활성화**: `--korean` 플래그가 있으면 registry 에 자동 추가 (다른 검사와 결합). Plan 의 결정은 **D1 (--korean opt-in, 자동 활성화)**.

## 3. 5개 규칙 (KS1-KS5)

### 3.1 정규식

```python
# KS1: 영어 단어 + 한글 (붙어있음)
# 예: "Python은" → "Python 은" 권장
# 매칭: \b영어단어(?=한글)
_KS1_RE = re.compile(r"[A-Za-z]+(?=[가-힣])")

# KS2: 한자 + 한글 (붙어있음)
# 예: "中文언어" → "中文 언어" 권장
_KS2_RE = re.compile(r"[一-鿿㐀-䶿]+(?=[가-힣])")

# KS3: 숫자 + 단위 띄어쓰기 (반대로, "10 개" 는 잘못)
# 예: "10 개" → "10개" (한국어 맞춤법, 단위 의존성 없이 단순)
# 매칭: \d + 공백 + 한글 단위 (조사 포함)
_KS3_RE = re.compile(r"\d+\s+[가-힣]+")

# KS4: 중복 공백
_KS4_RE = re.compile(r"[^\n ]{2,}| {2,}|[ \t]{2,}")

# KS5: 3개 연속 마침표
_KS5_RE = re.compile(r"\.{3,}")
```

### 3.2 메시지 (한국어)

| 규칙 | 메시지 |
|------|--------|
| KS1 | `영어 단어와 한글 사이 띄어쓰기 권장: 'Python은' (라인 N)` |
| KS2 | `한자와 한글 사이 띄어쓰기 권장: '中文언어' (라인 N)` |
| KS3 | `숫자와 한글 단위 사이 띄어쓰기 없음: '10 개' (라인 N)` |
| KS4 | `중복 공백 발견 (라인 N)` |
| KS5 | `'...' 대신 '…' 권장 (라인 N)` |

### 3.3 심각도

- **KS1, KS2, KS4, KS5**: INFO (권장)
- **KS3**: WARNING (한국어 맞춤법 위반)

## 4. 체크 — `checks/korean_grammar.py`

### 4.1 시그니처

```python
class KoreanGrammarCheck(BaseCheck):
    """한국어 띄어쓰기/맞춤법 검사 (opt-in, KS1~KS5)."""
    name = "korean-grammar"
    
    def run(self, context: dict[str, Any]) -> Iterator[Diagnosis]:
        # 코드 영역 제외
        lines = context["lines"]
        fences = mark_code_regions(lines)
        for i, line in enumerate(lines, start=1):
            if is_in_code_region(i, fences):
                continue
            yield from self._check_ks1(line, i)
            yield from self._check_ks2(line, i)
            yield from self._check_ks3(line, i)
            yield from self._check_ks4(line, i)
            yield from self._check_ks5(line, i)
```

### 4.2 각 규칙 함수

```python
def _check_ks1(line, line_no):
    for m in _KS1_RE.finditer(line):
        # 매치된 부분 = 영어 단어. 라인에서 그 단어 + 직후 한글 1자 표시.
        word = m.group(0)
        idx = m.end()
        if idx < len(line):
            next_char = line[idx]
            yield Diagnosis(
                check="korean-grammar",
                severity=Severity.INFO,
                message=f"영어 단어와 한글 사이 띄어쓰기 권장: '{word}{next_char}'",
                line=line_no,
                rule="KS1",
            )


def _check_ks2(line, line_no):
    # KS1 와 동일 패턴, 한자 + 한글
    ...


def _check_ks3(line, line_no):
    for m in _KS3_RE.finditer(line):
        yield Diagnosis(
            check="korean-grammar",
            severity=Severity.WARNING,
            message=f"숫자와 한글 단위 사이 띄어쓰기 없음: '{m.group(0)}'",
            line=line_no,
            rule="KS3",
        )


def _check_ks4(line, line_no):
    # 2+ 공백
    if "  " in line:
        yield Diagnosis(
            check="korean-grammar",
            severity=Severity.INFO,
            message=f"중복 공백 발견",
            line=line_no,
            rule="KS4",
        )


def _check_ks5(line, line_no):
    for m in _KS5_RE.finditer(line):
        yield Diagnosis(
            check="korean-grammar",
            severity=Severity.INFO,
            message="'...' 대신 '…' 권장",
            line=line_no,
            rule="KS5",
        )
```

## 5. CLI 통합

### 5.1 dispatch 로직

```python
def main(argv=None):
    args = parser.parse_args(argv)
    
    if args.korean:
        # registry 에 korean-grammar 추가
        if isinstance(registry := _filter_registry(args.checks.split(",") if args.checks else None), CheckRegistry):
            from md_doctor.checks.korean_grammar import KoreanGrammarCheck
            if not any(c.name == "korean-grammar" for c in registry.checks):
                registry.checks.append(KoreanGrammarCheck())
    
    # 기존 diagnose/fix 로직
    ...
```

**간단화**: registry 를 미리 만들고, --korean 플래그가 있으면 korean-grammar 를 append.

### 5.2 사용 예

```bash
# 한국어 검사만
md-doctor --korean README.md --checks korean-grammar

# 다른 검사와 결합
md-doctor --korean README.md

# 기존 diagnose 와 100% 호환 (--korean 없으면 비활성)
```

## 7. 엣지 케이스 / 실패 모드

| 케이스 | 처리 |
|--------|------|
| `Python` 단독 (한글 없음) | KS1 매칭 X (lookahead `(?=[가-힣])` 가 false) |
| `10` 단독 (한글 단위 없음) | KS3 매칭 X |
| `Python3.10` | KS1 매칭 (영어+숫자 = 영어 단어로 인식) |
| 코드펜스 내부 한글 | 무시 |
| 매우 긴 라인 (1000+자) | 정규식 O(n) — 무리 없음 |
| KS4 트레일링 공백 (v0.4.0 fix 로 제거됨) | KS4 는 `  ` (2+ 연속) 만. 트레일링 1개는 KS4 X |
