## 3. 데이터 모델 (신규)

### 3.1 `CodeFence` (신규, `extractors/code_fence.py`)

```python
@dataclass(frozen=True)
class CodeFence:
    """코드펜스 한 구간."""
    start_line: int
    end_line: int      # 닫는 줄 (없으면 None for 미닫힌)
    language: str = ""  # ```python → "python"
    marker: str = "```"  # ``` or ~~~
```

**사용**: 추출기에서 "이 줄이 코드 내부인가?" 빠르게 판별.

### 3.2 `HtmlRef` (신규, `extractors/html.py`)

```python
@dataclass(frozen=True)
class HtmlRef:
    """HTML 태그 한 개."""
    tag: str             # "img", "a", "div"
    line: int
    col: int
    attrs: dict[str, str]  # {"src": "x.png", "alt": "foo"}
    kind: str = "inline"   # "inline" | "block"
```

**사용**: `BrokenImagesCheck`/`DeadLinksCheck` 가 `<img src>`, `<a href>` 추출.

### 3.3 `Diagnosis` (변경 없음)

기존 모델 그대로. `check="gfm-lint"`, `rule="HC005"~"HC008"`. BI002/DL002는 기존 `broken-images`/`dead-links` check 이름 유지.

## 5. 추출기 — `extractors/html.py` (신규)

### 5.1 시그니처

```python
def extract_html_refs(
    text: str,
    lines: list[str],
    fences: list[CodeFence],
) -> list[HtmlRef]:
    """마크다운에서 HTML 태그를 추출. 코드펜스/인라인 코드 내부 제외.
    
    지원 태그 (v0.3.0 범위):
    - <img src="..." alt="..." />
    - <a href="...">text</a>
    - <div>, <p>, <span> 등 (HC007 만)
    """
```

### 5.2 정규식

```python
# 자체닫힘: <img src="x.png" alt="foo" />
_SELF_CLOSING_RE = re.compile(
    r'<([a-zA-Z][a-zA-Z0-9]*)\b([^>]*?)/?>',  # 마지막 /> 또는 >
)

# 쌍 태그: <a href="x">text</a>
_PAIRED_RE = re.compile(
    r'<([a-zA-Z][a-zA-Z0-9]*)\b([^>]*?)>(.*?)</\1>',
    re.DOTALL,
)

# 속성 파싱
_ATTR_RE = re.compile(
    r'([a-zA-Z][a-zA-Z0-9_-]*)\s*=\s*"([^"]*)"',
)
```

### 5.3 동작 규칙

| 입력 | 결과 |
|------|------|
| `<img src="x.png" alt="foo" />` | `HtmlRef("img", line, col, {"src": "x.png", "alt": "foo"}, "inline")` |
| `<a href="y.md">text</a>` | `HtmlRef("a", line, col, {"href": "y.md"}, "inline")` |
| `<div>content</div>` | `HtmlRef("div", line, col, {}, "block")` |
| `<br>` | (자체닫힘) `HtmlRef("br", ...)` |
| 코드펜스 내부 HTML | **스킵** |
| 인라인 코드 `<x>` | **스킵** |
| `<unknown>` 태그 | `HtmlRef("unknown", ...)` (HC007 대상) |

## 6. 체크 — `checks/gfm_lint.py` (신규)

### 6.1 시그니처

```python
class GfmLintCheck(BaseCheck):
    """GFM/CommonMark 규칙 린트 (HC005~008)."""
    name = "gfm-lint"
    
    def run(self, context: dict[str, Any]) -> Iterator[Diagnosis]:
        # 1. HC005: 리스트 점 일관성
        # 2. HC006: 코드언어 지정
        # 3. HC007: raw HTML
        # 4. HC008: 테이블 정렬
```

### 6.2 HC005 — 리스트 점 일관성 (WARNING)

```python
# 문서 내 글머리 기호 사용 추적
bullet_chars: set[str] = set()
ordered_styles: set[str] = set()  # "." or ")"

for line in lines:
    m_bullet = re.match(r"^(\s*)([-*+])", line)
    m_ordered = re.match(r"^(\s*)(\d+)([.)])", line)
    if m_bullet: bullet_chars.add(m_bullet.group(2))
    if m_ordered: ordered_styles.add(m_ordered.group(3))

if len(bullet_chars) > 1:
    yield Diagnosis(... rule="HC005", severity=WARNING,
        message=f"글머리 기호 혼용: {sorted(bullet_chars)}. 하나만 사용 권장.")
if len(ordered_styles) > 1:
    yield Diagnosis(... rule="HC005", severity=WARNING,
        message=f"순서 리스트 마커 혼용: {sorted(ordered_styles)}.")
```

### 6.3 HC006 — 코드언어 지정 (INFO)

```python
# 닫힌 코드펜스 중 언어가 없는 것
for fence in fences:
    if fence.end_line is not None and not fence.language:
        yield Diagnosis(... rule="HC006", severity=INFO,
            message=f"코드언어 미지정 (라인 {fence.start_line}). 신택스 하이라이팅 개선.")
```

### 6.4 HC007 — raw HTML (WARNING)

```python
# 모든 HTML 태그가 의심. <a>, <img>, <div> 등.
for ref in extract_html_refs(...):
    if ref.tag in ("img", "a"):
        # BI002/DL002 에서 별도 ERROR — HC007 은 WARNING 만
        yield Diagnosis(... rule="HC007", severity=WARNING,
            message=f"raw HTML <{ref.tag}> 사용. 마크다운 대체 권장.")
    elif ref.tag in ALLOWED_HTML:
        # <br>, <kbd>, <sub> 등 GFM 호환 → 스킵
        continue
    else:
        yield Diagnosis(... rule="HC007", severity=WARNING,
            message=f"raw HTML <{ref.tag}> 사용 (라인 {ref.line}). 호환성 주의.")
```

**`ALLOWED_HTML`**: `<br>`, `<kbd>`, `<sub>`, `<sup>`, `<details>`, `<summary>` (GFM 호환).

### 6.5 HC008 — 테이블 정렬 (WARNING)

```python
# GFM 테이블: 헤더 행 + separator 행 + 본문 행
# separator: |---|---| 또는 |:---:| 등
_SEPARATOR_RE = re.compile(r"^\s*\|?\s*:?-+:?\s*(\|\s*:?-+:?\s*)+\|?\s*$")

for i, line in enumerate(lines, start=1):
    if not _SEPARATOR_RE.match(line):
        continue
    # 직전 줄이 헤더
    if i < 2: continue
    header = lines[i - 2]
    n_cols_header = _count_columns(header)
    n_cols_sep = line.count("|") - 1 + (1 if line.startswith("|") else 0)
    # 더 정확한 계산은 row 파싱
    ...
```

**더 정밀한 파싱**:
```python
def _count_columns(row: str) -> int:
    # leading/trailing | 제외, | 로 split
    row = row.strip()
    if row.startswith("|"): row = row[1:]
    if row.endswith("|"): row = row[:-1]
    return len([c for c in row.split("|") if c.strip()])
```

## 9. 엣지 케이스 / 실패 모드

| 케이스 | 처리 |
|--------|------|
| 자체닫힘 `<br/>` vs `<br>` | 둘 다 매칭 (정규식이 `/?>`) |
| `<a href="x">text</a>` 멀티라인 | `re.DOTALL` 로 매칭 |
| 속성값에 `>` 포함 | 미지원 (v0.3.0 범위 외) |
| `<unknown>` 태그 | `tag="unknown"` 으로 추출 → HC007 |
| 미닫힌 코드펜스 | `end_line=None`, EOF까지 코드 |
| nested HTML (`<div><p>...</p></div>`) | 단순 정규식, flat 추출 |
