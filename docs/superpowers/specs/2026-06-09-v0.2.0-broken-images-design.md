## 2. 아키텍처

### 2.1 단일 책임 분리

```
src/md_doctor/
├── extractors/                  # [신규]
│   ├── __init__.py
│   └── images.py                # ImageRef, extract_image_refs()
├── checks/
│   ├── broken_images.py         # BrokenImagesCheck (수정: stub → 구현)
│   └── ...
└── ...
```

**왜 추출기를 분리하나?**
- `broken-images` 와 `dead-links` 가 **같은 `![](...)` 매칭**이 필요할 가능성 (향후)
- `extract_image_refs()` 는 Python API로도 노출 가치 (`from md_doctor.extractors.images import extract_image_refs`)
- 테스트가 좁아짐 — 추출기는 "어디서 무엇을 추출하나", 체크는 "추출한 것이 유효한가"

### 2.2 의존성 그래프

```
extractors/images.py   ← models (없음 — dataclass 만)
checks/broken_images.py ← extractors.images, models
checks/__init__.py      ← checks.broken_images (default_checks 에 등록)
core.py                 ← 변경 없음
cli.py                  ← 변경 없음
```

순환 import 없음. `extractors` 는 `models` 와 무관 (Path / dataclass 만 사용).

## 3. 데이터 모델

### 3.1 `ImageRef` (신규)

`src/md_doctor/extractors/images.py`:

```python
from __future__ import annotations

from dataclasses import dataclass


@dataclass(frozen=True)
class ImageRef:
    """마크다운에서 추출한 단일 이미지 참조."""
    
    target: str   # ![](images/foo.png) → "images/foo.png" (URL/경로만, alt 제외)
    line: int     # 1-based line 번호
    col: int      # 1-based column 번호 (디버깅용, 메시지에는 미포함)
    alt: str = "" # ![alt](url) → "alt", alt 없으면 ""
```

**왜 `frozen=True`?**
- `Diagnosis` 와 동일하게 immutable
- 해시 가능 → 향후 캐시 / set 연산 가능

### 3.2 `Diagnosis` (변경 없음)

기존 모델 그대로 사용. `check="broken-images"`, `rule="BI001"`, `severity=Severity.ERROR`.

## 4. 추출기 (`extractors/images.py`)

### 4.1 시그니처

```python
def extract_image_refs(
    text: str,
    lines: list[str],
) -> list[ImageRef]:
    """마크다운에서 ![](...) / ![alt](...) 참조를 추출.
    
    Parameters
    ----------
    text:
        원본 마크다운 텍스트.
    lines:
        ``text.splitlines()`` 와 동등한 줄 리스트.
    
    Returns
    -------
    list[ImageRef]
        코드펜스 / 인라인 코드 외부, 원격 URL / data: URL 제외한 참조.
    
    Notes
    -----
    - 정규식: ``r"!\[([^\]]*)\]\(([^)\s]+)(?:\s+\"[^\"]*\")?\)"`` (간이)
    - 한 줄에 여러 이미지가 있어도 모두 추출
    - 중복 참조도 그대로 모두 보고 (사용자에게 line으로 안내)
    """
```

### 4.2 동작 규칙

| 입력 | 결과 |
|------|------|
| `![](images/foo.png)` | `ImageRef("images/foo.png", line, col, alt="")` |
| `![Alt text](./x.png)` | `ImageRef("./x.png", line, col, alt="Alt text")` |
| `![](https://example.com/x.png)` | **스킵** (원격) |
| `![](data:image/png;base64,...)` | **스킵** (data:) |
| `` ![](x.png) `` (인라인 코드) | **스킵** |
| ``` ``` ![](x.png) ``` ``` (코드펜스) | **스킵** |
| `![]( images/spaced.png )` (공백) | `ImageRef(" images/spaced.png ", ...)` — **trim 안 함** (pathlib이 처리) |
| `![](x.png "title")` | `ImageRef("x.png", line, col, alt="")` (title 무시) |
| `<img src="x.png">` (HTML) | **스킵** (0.3.0+ 범위) |

### 4.3 알고리즘 의사코드

```python
def extract_image_refs(text, lines):
    in_code = False
    fence = None
    refs = []
    
    # 1단계: 코드펜스/인라인 코드 영역 마킹
    in_code_mask = _mark_code_regions(lines)  # list[bool], True=코드 내부
    
    # 2단계: 각 줄에서 ![](...) 찾기
    pattern = re.compile(r'!\[([^\]]*)\]\(([^)\s]+)(?:\s+"[^"]*")?\)')
    for i, line in enumerate(lines, start=1):
        if in_code_mask[i - 1]:
            continue
        for m in pattern.finditer(line):
            alt, target = m.group(1), m.group(2)
            if target.startswith(("http://", "https://", "data:", "mailto:")):
                continue
            refs.append(ImageRef(
                target=target,
                line=i,
                col=m.start() + 1,  # 1-based
                alt=alt,
            ))
    return refs
```

### 4.4 정규식 분해

- `!\[` — 리터럴 `![`
- `([^\]]*)` — alt 텍스트 (group 1, 비집약)
- `\]\(` — 리터널 `](`
- `([^)\s]+)` — target URL/경로 (group 2, 공백/닫는괄호 미포함)
- `(?:\s+"[^"]*")?` — 선택적 title (` ![](x "title")`)
- `\)` — 리터럴 `)`

**한계 (의도적)**: 
- `![alt with \[brackets\]](url)` → 매칭 실패 (의도적 단순화)
- 마크다운 spec의 모든 케이스를 다루지 않음 (0.1.x 범위에선 충분)
- 후속 버전에서 정밀 파서로 교체 가능 (현재 모듈 인터페이스가 안정)

## 5. 체크 (`checks/broken_images.py`)

### 5.1 시그니처

```python
class BrokenImagesCheck(BaseCheck):
    """`![](...)` 참조의 로컬 파일 유효성 검사.
    
    원격 URL(data:, http://, https://)은 스킵.
    `extract_image_refs()` 가 추출한 참조를 .md 파일 위치 기준으로
    ``Path.exists()`` 로 검증.
    """
    
    name = "broken-images"
    
    def run(self, context: dict[str, Any]) -> Iterator[Diagnosis]: ...
```

### 5.2 알고리즘

```python
def run(self, context):
    path: Path = context["path"]
    base_dir = path.parent
    refs = extract_image_refs(context["text"], context["lines"])
    
    for ref in refs:
        try:
            target = _resolve_path(base_dir, ref.target)
        except (ValueError, OSError) as e:
            yield Diagnosis(
                check=self.name,
                severity=Severity.ERROR,
                message=f"깨진 이미지 참조: 경로 해석 실패 '{ref.target}': {e}",
                line=ref.line,
                rule="BI001",
            )
            continue
        
        if not target.exists():
            yield Diagnosis(
                check=self.name,
                severity=Severity.ERROR,
                message=f"깨진 이미지 참조: '{ref.target}' (해상: {target})",
                line=ref.line,
                rule="BI001",
            )
            continue
        
        if target.is_dir():
            yield Diagnosis(
                check=self.name,
                severity=Severity.ERROR,
                message=f"깨진 이미지 참조: '{ref.target}' 은 디렉터리입니다 (파일이어야 함)",
                line=ref.line,
                rule="BI001",
            )
```

### 5.3 `_resolve_path()`

```python
def _resolve_path(base_dir: Path, target: str) -> Path:
    """base_dir 기준 상대경로 해석. 절대 경로는 그대로."""
    # pathlib이 공백/../ 처리
    p = Path(target)
    if p.is_absolute():
        return p
    return (base_dir / p).resolve()  # 정규화 (심볼릭 링크 X, .. 포함)
```

**`resolve()` 사용 이유**: `../` 정규화 → 사용자 메시지에 깔끔한 절대 경로 표시.

## 7. 엣지 케이스 / 실패 모드

| 케이스 | 처리 |
|--------|------|
| 이미지 경로가 매우 김 (>4096) | `OSError` 가능 → try/except로 잡아 ERROR 진단 |
| `Path.resolve()` 가 없는 경로 (Windows) | `strict=False` (기본값) 사용 |
| 권한 없음 (PermissionError) | `Path.exists()` 가 False 반환 → ERROR 진단 |
| 깨진 심볼릭 링크 | `Path.exists()` 가 False → ERROR 진단 |
| 동시 다발적 깨짐 (10+개) | 그대로 모두 보고 (사용자가 일괄 수정 가능) |
| `.md` 파일 자체가 깨짐 (read 오류) | `core._read_markdown` 가 `UnreadableFileError` → core 단계에서 처리, 본 검사 도달 X |
| 정규식 매칭 miss (e.g. `![alt with [brackets]](url)`) | 0.2.0 범위 외 — INFO 진단도 없음 (의도적) |

## 8. 에러 처리

| 단계 | 예외 | 처리 |
|------|------|------|
| 파일 읽기 | `UnreadableFileError` | core 단계, 본 검사 미도달 |
| 경로 해석 | `OSError`, `ValueError` | ERROR 진단 |
| `Path.exists()` | 예외 없음 (bool 반환) | — |
| `Path.is_dir()` | 예외 없음 | — |

모든 사용자 메시지는 **한국어** (기존 코드 스타일 일관). 진단 메시지 prefix:
- `"깨진 이미지 참조: 'TARGET' (해상: RESOLVED)"`
- `"깨진 이미지 참조: 'TARGET' 은 디렉터리입니다 (파일이어야 함)"`
- `"깨진 이미지 참조: 경로 해석 실패 'TARGET': REASON"`

## 9. 성능

- `Path.exists()` 는 stat 1회 (보통 < 1ms)
- 한 파일당 100개 이미지 참조 → 100 stat → ~100ms (오프라인, 로컬 SSD)
- 1000개 파일 트리 × 평균 5 이미지 = 5000 stat → ~5s
- 캐시: v0.2.0 범위 외 (심볼릭 링크 무효화 이슈). v0.3.0+ 에서 `stat` 결과 캐시.

**벤치마크 목표**: 1000개 .md 파일 트리, 각 10개 이미지 = ~10s. baseline 측정 후 회귀 가드.
