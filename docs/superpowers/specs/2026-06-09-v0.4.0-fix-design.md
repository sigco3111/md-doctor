## 2. 아키텍처

### 2.1 모듈 구조

```
src/md_doctor/
├── fixers/                    # [신규]
│   ├── __init__.py            # apply_fixes(), list_fixes()
│   └── transforms.py          # 6개 변환 함수
└── cli.py                     # [수정] `fix` subcommand 추가

**왜 `fixers/` 별도 패키지?**
- 단일 책임: 변환 로직은 lint 와 독립
- 6개 함수가 한 모듈에 모이므로 `transforms.py` 단일 파일
- Python API 로도 노출 가치 (`from md_doctor.fixers import apply_fixes`)

### 2.2 의존성 그래프

```
fixers/transforms.py   ← models (없음)
fixers/__init__.py     ← transforms (re-export)
cli.py                 ← fixers (subcommand 등록)
```

순환 없음. lint 와 독립.

### 2.3 새 CLI subcommand 구조

`cli.py` 의 `main()` 함수가 `args.command` 에 따라 `diagnose` 또는 `fix` 를 dispatch:

```python
parser = argparse.ArgumentParser(...)
subparsers = parser.add_subparsers(dest="command")

# 기존 (default): md-doctor <path>
# argparse: "command" 미지정 시 None → 기본값 "diagnose"

# 신규: md-doctor fix <path>
fix_parser = subparsers.add_parser("fix", help="자동 후처리")
fix_parser.add_argument("path", type=Path)
fix_parser.add_argument("--backup", default=".bak", help="백업 접미사 (기본: .bak)")
fix_parser.add_argument("--no-backup", action="store_true")
fix_parser.add_argument("--dry-run", action="store_true")
fix_parser.add_argument("--format", choices=["text", "json"], default="text")
```

**호환성**: 기존 `md-doctor <path>` 사용자는 `command` 미지정 → `diagnose` 가 default → 기존 동작 100% 보존.

## 4. 변환 (F1~F6) — `fixers/transforms.py`

### 4.1 6개 변환 함수

각 함수는 `(text: str) -> tuple[str, bool]` — (변환된 텍스트, 변경 여부).

```python
def f1_trailing_newline(text: str) -> tuple[str, bool]:
    """F1: 파일 끝 newline 1개 보장."""
    if text.endswith("\n"):
        return text, False
    if not text:
        return text, False
    return text + "\n", True
```

```python
def f2_trailing_whitespace(text: str) -> tuple[str, bool]:
    """F2: 라인 끝 공백 제거."""
    lines = text.split("\n")
    new_lines = [line.rstrip() for line in lines]
    new_text = "\n".join(new_lines)
    return new_text, new_text != text
```

```python
def f3_heading_blank_line(text: str) -> tuple[str, bool]:
    """F3: H1/H2 헤딩 다음 빈 줄 보장.
    
    ```python
    # H1
    본문
    ```
    → 
    ```python
    # H1
    
    본문
    ```
    """
    lines = text.split("\n")
    new_lines: list[str] = []
    for i, line in enumerate(lines):
        new_lines.append(line)
        # 헤딩이고 다음 줄이 비어있지 않으면 빈 줄 추가
        if re.match(r"^#{1,2}\s+", line) and i + 1 < len(lines) and lines[i + 1].strip():
            new_lines.append("")
    new_text = "\n".join(new_lines)
    return new_text, new_text != text
```

```python
def f4_empty_heading(text: str) -> tuple[str, bool]:
    """F4: 빈 헤딩 (e.g. '## ') 제거."""
    lines = text.split("\n")
    new_lines = [line for line in lines if not re.match(r"^#{1,6}\s*$", line)]
    new_text = "\n".join(new_lines)
    return new_text, new_text != text
```

```python
def f5_strip_bom(text: str) -> tuple[str, bool]:
    """F5: UTF-8 BOM 제거."""
    if text.startswith("\ufeff"):
        return text[1:], True
    return text, False
```

```python
def f6_crlf_to_lf(text: str) -> tuple[str, bool]:
    """F6: CRLF → LF."""
    if "\r\n" in text:
        return text.replace("\r\n", "\n"), True
    return text, False
```

### 4.2 변환 체인

```python
ALL_TRANSFORMS: list[tuple[str, Callable[[str], tuple[str, bool]]]] = [
    ("F1", f1_trailing_newline),
    ("F2", f2_trailing_whitespace),
    ("F3", f3_heading_blank_line),
    ("F4", f4_empty_heading),
    ("F5", f5_strip_bom),
    ("F6", f6_crlf_to_lf),
]


def apply_fixes(
    text: str,
    *,
    enabled: set[str] | None = None,  # None = all
) -> FixResult:
    """모든 enabled 변환을 순서대로 적용. 변경 추적."""
    enabled_ids = enabled or {fid for fid, _ in ALL_TRANSFORMS}
    current = text
    applied: list[str] = []
    changed_lines: set[int] = set()
    
    for fid, transform in ALL_TRANSFORMS:
        if fid not in enabled_ids:
            continue
        new_text, changed = transform(current)
        if changed:
            # 변경된 라인 추적 (간이: split 비교)
            _track_changed_lines(current, new_text, changed_lines)
            applied.append(fid)
        current = new_text
    
    return FixResult(
        original=text,
        fixed=current,
        changes=applied,
        changed_lines=sorted(changed_lines),
    )
```

### 4.3 변환 순서가 중요한 이유

- F1 (newline) 을 F6 (CRLF) 이후에 적용해야 LF 이미 있을 때 또 추가 안 함
- F5 (BOM) 을 가장 먼저 적용 (다른 변환이 BOM 영향 받지 않도록)
- F4 (빈 헤딩) 를 F3 (헤딩 후 빈 줄) 이전에 적용 (F4 가 헤딩 줄 자체를 제거하면 F3 가 처리할 헤딩이 줄어듦)

Plan의 순서: F1 → F2 → F3 → F4 → F5 → F6. 이 순서로:
- F5 (BOM) 가 마지막에 오면 변환 추적이 깨짐 (splitlines 가 BOM 영향). F5 를 마지막에 두면 라인 추적 어려움. **F5 를 앞 순서로 변경**.

수정: **F5 → F1 → F2 → F3 → F4 → F6** (BOM 먼저, CRLF 마지막)

## 5. CLI — `md-doctor fix`

### 5.1 시그니처

```python
def fix_file(
    path: Path,
    *,
    backup_suffix: str = ".bak",
    no_backup: bool = False,
    dry_run: bool = False,
) -> FixResult:
    """1개 파일을 fix. 결과 + (in-place 또는 dry-run).
    
    - no_backup=False, dry_run=False: 백업 생성 + in-place 덮어쓰기
    - dry_run=True: 원본 보존, FixResult 만 반환
    - no_backup=True: 백업 생략, in-place 덮어쓰기
    """
```

### 5.2 출력 형식

**text**:
```
🔧 md-doctor v0.4.0 fix — /path/to/file.md
   ✅ 변경 4건 (F1, F2, F3, F6)
   - 라인 12: trailing whitespace 제거
   - 라인 30: heading 뒤 빈 줄 추가
   - 라인 31: trailing newline 추가
   (변경 0건이면 "✅ 변경 없음")
```

**json**:
```json
{
  "version": "0.4.0",
  "file": "/path/to/file.md",
  "dry_run": false,
  "applied": ["F1", "F2", "F3", "F6"],
  "changed_lines": [12, 30, 31],
  "before": "...",
  "after": "..."
}
```

### 5.3 백업

```python
if not no_backup and not dry_run:
    backup_path = path.with_suffix(path.suffix + backup_suffix)
    shutil.copy2(path, backup_path)  # 메타데이터 보존
```

- 기본: `README.md` → `README.md.bak`
- `--backup .orig` → `README.md.orig`
- `--no-backup` → 백업 생략

### 5.4 트리 fix (확장, v0.4.1+)

`md-doctor fix docs/` (디렉터리) → v0.4.0 에서는 단일 파일만 지원. 디렉터리는 v0.4.1+.

## 7. 엣지 케이스 / 실패 모드

| 케이스 | 처리 |
|--------|------|
| 빈 파일 | 변환 0건, FixResult 빈 changes |
| UTF-8 디코딩 실패 | FixError (예외) 또는 skip — `errors='replace'` (안전) |
| 매우 큰 파일 (100MB) | 1-pass 변환, 메모리 OK |
| LF 와 CRLF 혼재 | F6 가 모든 `\r\n` → `\n` 변환 |
| 빈 헤딩에 trailing space | F2 → F4 순서로 둘 다 처리 |
| 헤딩 + 코드펜스 | 코드펜스 내부 헤딩은 추출기 패턴과 무관, F3 가 그대로 적용. 위험. **v0.4.0 범위 외: 코드펜스 인식**. |
| 파일이 디렉터리 | `IsADirectoryError` (cli 가 잡아 stderr) |
