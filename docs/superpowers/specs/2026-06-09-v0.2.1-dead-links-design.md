## 3. 데이터 모델

### 3.1 `LinkRef` (신규)

`src/md_doctor/extractors/links.py`:

```python
from __future__ import annotations

from dataclasses import dataclass


@dataclass(frozen=True)
class LinkRef:
    """마크다운에서 추출한 단일 링크 참조."""
    
    target: str
    line: int
    col: int
    text: str = ""
    kind: str = "inline"  # "inline" | "reference" | "shortcut" | "autolink" | "definition"
```

**왜 `kind`?**
- 사용자에게 어떤 형태의 링크가 깨졌는지 알림 (INFO 메시지 향상)
- 추후 정의 자체는 ERROR 대상이 아니므로 구분

### 3.2 `Diagnosis` (변경 없음)

기존 모델 그대로 사용. `check="dead-links"`, `rule="DL001"`, `severity=Severity.ERROR`.

## 4. 추출기 (`extractors/links.py`)

### 4.1 시그니처

```python
def extract_link_refs(text: str, lines: list[str]) -> list[LinkRef]:
    """마크다운에서 모든 링크 참조를 추출.
    
    Parameters
    ----------
    text:
        원본 마크다운 텍스트.
    lines:
        ``text.splitlines()`` 와 동등한 줄 리스트.
    
    Returns
    -------
    list[LinkRef]
        인라인/참조/정의 링크 + 자동 링크, 코드 영역 제외.
    
    Notes
    -----
    - 정규식 + 2-pass (1차 정의 수집, 2차 참조 추출)
    - 한 줄에 여러 링크 모두 추출
    - 중복 정의는 마지막 것 사용
    """
```

### 4.2 동작 규칙

| 입력 | kind | 결과 |
|------|------|------|
| `[text](url)` | inline | `LinkRef("url", line, col, "text", "inline")` |
| `[text](url "title")` | inline | title 무시 |
| `[text][ref]` | reference | 정의 테이블에서 `[ref]: url` 찾음 |
| `[text][]` | reference | 텍스트가 ref, 정의 테이블에서 `[text]: url` 찾음 |
| `[ref]` | shortcut | 정의 테이블에서 `[ref]: url` 찾음 |
| `[ref]: url` | definition | (단독 진단 X, 참조에서 사용) |
| `<https://...>` | autolink | `LinkRef("https://...", line, col, "", "autolink")` |
| `<foo@bar.com>` | autolink | `mailto:foo@bar.com` 으로 간주 (스킵) |
| `mailto:...` (target) | - | **스킵** (브레인스토밍 결정) |
| `data:...` (target) | - | **스킵** |
| `javascript:...` (target) | - | **스킵** |
| `#fragment-only` (target) | - | **스킵** (외부 도메인 검사 의미 없음) |
| 코드펜스 내부 | - | **스킵** |
| 인라인 코드 내부 | - | **스킵** |

### 4.3 알고리즘 의사코드

```python
def extract_link_refs(text, lines):
    # 1단계: 정의 테이블 수집 (kind="definition", 별도 리스트)
    definitions = _parse_definitions(text)  # {ref_id: url}
    
    # 2단계: 본문 스캔 (코드 영역 제외)
    refs = []
    in_code = False
    fence_marker = None
    
    for i, line in enumerate(lines, start=1):
        # 코드펜스 토글 (v0.2.0과 동일 로직)
        ...
        
        # 인라인 코드 마스킹
        masked = _strip_inline_code(line)
        
        # 인라인 링크 [text](url)
        for m in _INLINE_RE.finditer(masked):
            text_, url = m.group(1), m.group(2)
            if _is_skippable_url(url):
                continue
            refs.append(LinkRef(url, i, m.start() + 1, text_, "inline"))
        
        # 참조 링크 [text][ref], [text][], [ref]
        for m in _REFERENCE_RE.finditer(masked):
            text_, ref_id = m.group(1), m.group(2) or m.group(1)
            if ref_id in definitions:
                url = definitions[ref_id]
                if _is_skippable_url(url):
                    continue
                kind = "reference" if m.group(2) else "shortcut"
                refs.append(LinkRef(url, i, m.start() + 1, text_, kind))
        
        # 자동 링크 <url>
        for m in _AUTOLINK_RE.finditer(masked):
            url = m.group(1)
            if _is_skippable_url(url):
                continue
            refs.append(LinkRef(url, i, m.start() + 1, "", "autolink"))
    
    return refs
```

### 4.4 정규식 분해

```python
# 인라인 링크: [text](url "title")
_INLINE_RE = re.compile(
    r'\[([^\]]+)\]\(([^)\s]+)(?:\s+"[^"]*")?\)'
)

# 참조 링크: [text][ref], [text][] (ref 생략 → text 사용)
_REFERENCE_RE = re.compile(
    r'\[([^\]]+)\](?:\[([^\]]*)\])?'
)

# 자동 링크: <https://...> 또는 <foo@bar.com>
_AUTOLINK_RE = re.compile(
    r'<((?:https?|mailto|data|ftp)://[^>\s]+|[^>\s]+@[^>\s]+)>'
)

# 정의: [ref]: url "title" (라인 단위로 매칭)
_DEFINITION_RE = re.compile(
    r'^\s{0,3}\[([^\]]+)\]:\s+(\S+)(?:\s+"[^"]*")?$'
)
```

## 5. HEAD 요청 함수 (`checks/dead_links.py`)

### 5.1 시그니처

```python
def head_request(url: str, timeout: float = 5.0) -> int:
    """urllib HEAD 요청 → HTTP status code 반환.
    
    - 405 (Method Not Allowed) 또는 HEAD 미지원 → GET fallback
    - User-Agent 명시 (일부 서버는 default UA 차단)
    - redirect 자동 처리 (urllib 기본)
    
    Raises
    ------
    URLError
        DNS 실패, 연결 거부, 타임아웃.
    HTTPError
        status >= 400 (caller 가 처리).
    """
```

### 5.2 구현

```python
import urllib.request
import urllib.error

UA = "md-doctor/0.2.1 (+https://github.com/sigco3111/md-doctor)"

def head_request(url: str, timeout: float = 5.0) -> int:
    req = urllib.request.Request(url, method="HEAD", headers={"User-Agent": UA})
    try:
        with urllib.request.urlopen(req, timeout=timeout) as resp:
            return resp.status
    except urllib.error.HTTPError as e:
        # HEAD 자체가 4xx/5xx → 그대로 status 반환
        return e.code
    except urllib.error.URLError:
        # DNS 실패, timeout, 연결 거부 → caller 가 ERROR 진단
        raise
```

### 5.3 HEAD 미지원 fallback (YAGNI)

일부 서버는 HEAD를 405 로 거부하지만 GET 은 가능. v0.2.1 에서는 **405 → ERROR** 로 처리 (단순함). v0.3.0+ 에서 GET fallback 추가 검토.

### 6.1 시그니처

```python
    def run(self, context: dict[str, Any]) -> Iterator[Diagnosis]: ...
```

### 6.2 알고리즘

```python
def run(self, context):
    path: Path = context["path"]
    base_dir = path.parent
    refs = extract_link_refs(context["text"], context["lines"])
    
    for ref in refs:
        # 정의 자체는 진단 대상 아님 (참조에서 사용됨)
        if ref.kind == "definition":
            continue
        if _is_skippable_url(ref.target):
            continue
        
        if _is_local_path(ref.target):
            yield from _check_local(base_dir, ref)
        else:
            yield from _check_remote(ref)


def _check_local(base_dir, ref) -> Iterator[Diagnosis]:
    try:
        target = _resolve_local(base_dir, ref.target)
    except (ValueError, OSError) as e:
        yield Diagnosis(
            check="dead-links", severity=Severity.ERROR,
            message=f"깨진 링크: 경로 해석 실패 '{ref.target}': {e}",
            line=ref.line, rule="DL001",
        )
        return
    
    if not target.exists():
        yield Diagnosis(
            check="dead-links", severity=Severity.ERROR,
            message=f"깨진 링크: '{ref.target}' (해상: {target})",
            line=ref.line, rule="DL001",
        )
        return
    
    if target.is_dir():
        yield Diagnosis(
            check="dead-links", severity=Severity.ERROR,
            message=f"깨진 링크: '{ref.target}' 은(는) 디렉터리입니다",
            line=ref.line, rule="DL001",
        )


def _check_remote(ref) -> Iterator[Diagnosis]:
    try:
        status = head_request(ref.target, timeout=5.0)
    except (urllib.error.URLError, TimeoutError, OSError) as e:
        yield Diagnosis(
            check="dead-links", severity=Severity.ERROR,
            message=f"데드 링크: '{ref.target}' (연결 실패: {e.reason if hasattr(e, 'reason') else e})",
            line=ref.line, rule="DL001",
        )
        return
    
    if status >= 400:
        yield Diagnosis(
            check="dead-links", severity=Severity.ERROR,
            message=f"데드 링크: '{ref.target}' (HTTP {status})",
            line=ref.line, rule="DL001",
        )
    # 2xx/3xx → OK (진단 없음)


def _is_local_path(url: str) -> bool:
    """http(s):// 또는 다른 스킴이 없으면 로컬로 간주."""
    return "://" not in url


def _is_skippable_url(url: str) -> bool:
    """검사하지 않는 URL 스킴/패턴."""
    if url.startswith(("mailto:", "data:", "javascript:", "tel:")):
        return True
    if url.startswith("#"):  # fragment-only
        return True
    return False


def _resolve_local(base_dir: Path, target: str) -> Path:
    """로컬 경로 해석. fragment(#x) 제거 후 Path 처리."""
    # fragment 분리
    target_no_frag = target.split("#", 1)[0]
    if not target_no_frag:
        # fragment-only 는 호출자에서 이미 스킵
        raise ValueError("empty path")
    p = Path(target_no_frag)
    if p.is_absolute():
        return p
    return (base_dir / p).resolve()
```

```python
    (tmp_path / "b.md").write_text("[OK](ok.md)\n", encoding="utf-8")
    report = diagnose_tree(tmp_path)
    dl_by_file = {
        f.path.name: [d for d in f.diagnoses if d.check == "dead-links"]
        for f in report.files
    }
    assert len(dl_by_file["a.md"]) == 1
    assert dl_by_file["b.md"] == []
```

## 8. 엣지 케이스 / 실패 모드

| 케이스 | 처리 |
|--------|------|
| DNS 실패 (`URLError`) | ERROR 진단 |
| 타임아웃 (5s) | ERROR 진단 (reason 포함) |
| 405 Method Not Allowed | ERROR (v0.2.1) — GET fallback 미지원 |
| redirect chain | urllib 자동 처리 (3xx → 따라감) |
| 인증 필요 (401/403) | ERROR 진단 |
| 로컬 fragment-only (`#x`) | `_is_skippable_url` 에서 스킵 |
| 로컬 + fragment (`foo.md#section`) | fragment 제거 후 파일 존재 검사 |
| 매우 긴 URL | 그대로 검사 (timeout만 의존) |
| HTTP URL (`http://...`) | 검사 (HTTPS 우선 안 함) |
| FTP URL | 자동링크는 매칭, head_request 시도 → URLError 가능 |
| IPv6/도메인 punycode | urllib 자동 처리 |
| 응답 본문 100MB+ | HEAD 는 본문 없음 (안전) |
| 서버 다운 | URLError (Connection refused) → ERROR |
| self-signed SSL | urllib 기본 = verify (안전) |

## 9. 에러 처리

| 단계 | 예외 | 처리 |
|------|------|------|
| `head_request` DNS 실패 | `URLError` | ERROR (reason) |
| `head_request` timeout | `URLError` 또는 `TimeoutError` | ERROR |

- `"깨진 링크: 'TARGET' 은(는) 디렉터리입니다"`
- `"데드 링크: 'TARGET' (HTTP STATUS)"`
- `"데드 링크: 'TARGET' (연결 실패: REASON)"`

## 10. 성능

- **로컬 검사**: Path.stat 1회 (오프라인, < 1ms/파일)
- **원격 검사** (캐시 없음, 동기 순차):
  - 1 URL × 평균 0.3s (정상) = 0.3s
  - 1 URL × 5s (모두 타임아웃) = 5s
  - 100 URL × 평균 0.3s = 30s (보통)
  - 100 URL × 5s (최악) = 500s
- **권장 사항** (CLI 옵션, v0.3.0+ 후보):
  - `--max-urls N`: 기본 무제한, N 초과 시 처음 N개만 검사
  - `--skip-network`: 원격 URL 스킵 (로컬만 검사)
- v0.2.1 범위: 위 옵션 미포함 (단순함 우선)
